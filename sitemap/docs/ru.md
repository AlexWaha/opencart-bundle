## Информация о модуле

**Unlimited Sitemap** генерирует XML-карту сайта для OpenCart, которая работает на очень больших каталогах. Стандартная карта сайта OpenCart падает на крупных магазинах, потому что во время генерации ресайзит каждое изображение товара; этот модуль никогда не ресайзит при генерации и разбивает карту на шарды (`sitemapindex` + дочерние файлы), поэтому продолжает работать на каталогах в 50 000+ товаров.

### Ключевые возможности

- **Два режима генерации** - динамический (по запросу, один файл) и статический (cron, шардированные файлы на диске)
- **Шардинг** - большие каталоги разбиваются на несколько файлов карты, на которые ссылается `sitemapindex`
- **Безопасность кэша картинок** - изображения товаров добавляются только если ресайз уже лежит в `image/cache`; модуль не запускает ресайз во время генерации
- **Провайдеры-адаптеры** - каждый раздел карты это небольшой drop-in файл, поэтому вы можете добавить свои сущности (блог, новости, лендинги), не трогая ядро
- **Мультиязычность** - генерирует отдельный индекс и шарды для каждого языка магазина
- **Опциональный кэш** - динамический режим может кэшировать сгенерированный XML с настраиваемым TTL
- **Помощник для robots.txt** - показывает готовую строку `Sitemap:` и сниппеты серверного rewrite

## Инструкция по установке

### Шаг 1: Предварительные требования

1. Убедитесь, что установлен модуль **[aw_core_oc2.3-3.x.ocmod.zip](https://github.com/AlexWaha/opencart-bundle/blob/main/Core/dist/aw_core_oc2.3_3.x.ocmod.zip)**.

   > **Важно:** это вспомогательный функционал, необходимый всем модулям AlexWaha.

2. Для OpenCart 2.3.x убедитесь, что отключена FTP-загрузка расширений (или указаны корректные FTP-настройки магазина).

   > **Примечание:** для OpenCart 3.x это не требуется.

### Шаг 2: Установка модуля

1. Установите архив **aw_sitemap_oc2.3-3.x.ocmod.zip** через админ-панель:
   - Перейдите в **Расширения → Установка расширений**
   - Загрузите архив модуля
   - Дождитесь завершения установки

2. Обновите кэш модификаций:
   - Перейдите в **Расширения → Модификации**
   - Нажмите кнопку **Обновить**

3. В разделе **Расширения → Ленты данных** найдите и включите модуль:

   **alexwaha.com - Unlimited Sitemap**

   > **Важно:** при включении модуль устанавливает необходимые права доступа.

### Шаг 3: Самопроверка после установки

- ✓ Модуль присутствует в списке лент данных
- ✓ Страница настроек открывается без ошибок
- ✓ Вкладка **Провайдеры** показывает встроенные разделы (Товары, Категории, Производители, Информационные страницы, Главная, Лендинги)

## Основные настройки

Откройте настройки модуля: **Расширения → Ленты данных → alexwaha.com - Unlimited Sitemap**

### Статус
- **Включено** - модуль активен
- **Отключено** - генерация заблокирована

### Режим генерации
- **Динамический (по запросу)** - карта строится при каждом запросе и отдаётся одним файлом. Подходит для малых каталогов.
- **Статический (cron, шардинг)** - карта пишется на диск шардированными файлами через cron-команду. Подходит для больших каталогов.

> Если выбран Динамический режим и число товаров превышает заданный лимит, на странице показывается предупреждение с рекомендацией перейти на Статический режим.

### Папка вывода
Публичная папка в корне магазина, куда пишутся статические файлы.

**По умолчанию:** `sitemap`

**Требования:** 2-64 символа, буквы, цифры, дефис, подчёркивание. Создаётся автоматически при первой генерации.

### URL на один файл
Максимальное число URL в одном файле карты в статическом режиме.

**По умолчанию:** 5000 · **Диапазон:** 100 - 50000

> Протокол sitemap допускает до 50 000 URL на файл. 5 000 сохраняет каждый файл небольшим и быстрым в отдаче.

### Изображения товаров
Переключатель, включать ли записи `<image:image>` для товаров. Добавляются только картинки, уже закэшированные в `image/cache`. См. [Изображения товаров](#изображения-товаров).

### Языки
Языки для генерации. Оставьте все снятыми, чтобы генерировать все языки магазина.

### Кэш (динамический режим)
Если включено, сгенерированный XML кэшируется в динамическом режиме и отдаётся до истечения времени жизни.

### Время жизни кэша, сек
Сколько динамический кэш считается актуальным. **По умолчанию:** 3600

### Лимит товаров для динамического режима
Порог для предупреждения в Динамическом режиме. **По умолчанию:** 1000

## Провайдеры

Откройте вкладку **Провайдеры**, чтобы увидеть каждый раздел, добавляющий URL в карту. Каждый провайдер можно включить или отключить.

Встроенные провайдеры:

| Провайдер | Код | Какие URL добавляет |
|---|---|---|
| Товары | `product` | страницы `product/product` (шардинг) |
| Категории | `category` | страницы `product/category` |
| Производители | `manufacturer` | страницы `product/manufacturer/info` |
| Информационные страницы | `information` | страницы `information/information` |
| Главная | `home` | главная страница магазина |
| Лендинги | `landing` | страницы `aw_landing_page` (только если установлен модуль Landing Pages) |

Провайдеры обнаруживаются автоматически: любой файл, положенный в папку провайдеров, появляется здесь. См. [Создание провайдера](#для-разработчиков---создание-провайдера).

## Режимы генерации

### Динамический режим

Карта генерируется при запросе маршрута:

```
https://ваш-сайт.com/index.php?route=extension/aw_sitemap/sitemap
```

Возвращается один `<urlset>` со всеми URL включённых провайдеров. Опционально кэшируется в файл с TTL. В этом режиме физического файла `sitemap.xml` нет - чтобы отдавать чистый URL `/sitemap/sitemap.xml`, нужен серверный rewrite (см. ниже).

### Статический режим

Запустите cron-команду (или откройте маршрут один раз). Модуль пишет:

```
sitemap/sitemap.xml          <- индекс (sitemapindex)
sitemap/en-gb-product-0.xml  <- шард
sitemap/en-gb-product-1.xml
sitemap/en-gb-category-0.xml
...
```

Индекс ссылается на каждый шард по его реальному публичному URL. Файлы перегенерируются при каждом запуске. Это рекомендованный режим для больших каталогов.

---

## Статическая генерация через Cron

Добавьте CLI-команду со вкладки **Cron и robots.txt** в cron сервера, например раз в сутки:

```cron
0 3 * * * php /home/user/public_html/cli/aw_sitemap.php
```

Команда также показана в админке готовой к копированию. После её выполнения индекс карты доступен по адресу:

```
https://ваш-сайт.com/sitemap/sitemap.xml
```

Скопируйте строку `Sitemap:` с той же вкладки в `robots.txt`:

```
Sitemap: https://ваш-сайт.com/sitemap/sitemap.xml
```

> **Рекомендация:** генерируйте один-два раза в сутки. Чаще не нужно, если каталог не меняется постоянно.

## Серверный rewrite (динамический режим)

В **динамическом** режиме физического файла нет, поэтому `/sitemap/sitemap.xml` на диске не существует. Чтобы отдавать карту по этому чистому URL, добавьте правило rewrite (оба сниппета показаны готовыми к копированию на вкладке Cron):

**Apache (.htaccess):**
```apache
RewriteRule ^sitemap/sitemap.xml$ index.php?route=extension/aw_sitemap/sitemap [L]
```

**Nginx:**
```nginx
location = /sitemap/sitemap.xml {
    try_files $uri /index.php?route=extension/aw_sitemap/sitemap;
}
```

> **Статическому режиму rewrite не нужен** - он пишет реальный файл, который веб-сервер отдаёт напрямую.

## Изображения товаров

Это та функция, которая сохраняет стабильность модуля на больших каталогах.

Когда **Изображения товаров** включены, каждый товар может содержать запись `<image:image>`. Модуль ищет ресайз-миниатюру в `image/cache`:

- **Если кэшированная миниатюра существует** - её URL добавляется в карту.
- **Если её нет** - изображение пропускается. Модуль не вызывает ресайзер OpenCart во время генерации.

Это важно, потому что ресайз десятков тысяч изображений в одном запросе - именно то, из-за чего штатная карта OpenCart падает по таймауту или памяти. Используя только уже закэшированные миниатюры, генерация остаётся быстрой и предсказуемой.

> **Совет:** миниатюры товаров создаются естественно, пока покупатели просматривают каталог, либо вы можете прогреть кэш любым способом предварительного создания миниатюр. Ещё не закэшированные картинки просто попадут в карту при следующем запуске.

## Для разработчиков - создание провайдера

Провайдер - это небольшой контроллер каталога, добавляющий один раздел в карту. Ядро берёт на себя весь шардинг и циклы; провайдер только возвращает строки URL.

### Расположение и именование

```
catalog/controller/extension/aw_sitemap/provider/<name>.php
```

Файл `blog.php` -> класс `ControllerExtensionAwSitemapProviderBlog`.
Файл `landing_page.php` -> класс `ControllerExtensionAwSitemapProviderLandingPage`.

Файлы с префиксом `_` (например `_example.php`) - шаблоны и пропускаются.

### Контракт

```php
class ControllerExtensionAwSitemapProviderBlog extends Controller
{
    public function getCode(): string   { return 'blog'; }
    public function getName(): string   { return 'Blog posts'; }

    public function getTotal(int $languageId): int
    {
        // общее число URL для шардинга
        return 0;
    }

    public function getUrls(int $languageId, int $start, int $limit): array
    {
        // вернуть срез строк URL
        return [];
    }
}
```

### Формат строки URL

```php
[
    'loc'        => 'https://store/...',          // обязательно, абсолютный URL
    'lastmod'    => '2026-06-16T12:00:00+00:00',  // ISO 8601, опционально
    'changefreq' => 'weekly',                     // опционально
    'priority'   => '0.5',                        // опционально
    'images'     => [                             // опционально
        ['loc' => '...', 'caption' => '...', 'title' => '...'],
    ],
]
```

`priority` и `changefreq` задаются в коде провайдера, а не в админке. Готовый шаблон лежит в `catalog/controller/extension/aw_sitemap/provider/_example.php`, а рабочий пример на модуле Landing Pages - в `provider/landing.php`.

Как только файл на месте, он автоматически появляется на вкладке **Провайдеры**, где его можно включить или отключить.

## Возможные ошибки и рекомендации

### Карта пустая или нет разделов
- Проверьте, что нужные провайдеры включены на вкладке **Провайдеры**.
- Убедитесь, что товары/категории включены и привязаны к текущему магазину.

### `/sitemap/sitemap.xml` отдаёт 404 в динамическом режиме
- В динамическом режиме физического файла нет. Либо переключитесь на **Статический** режим (рекомендуется для больших каталогов), либо добавьте серверный rewrite со вкладки Cron.

### Статические файлы не пишутся
- Проверьте, что веб-сервер может писать в корень магазина для создания папки вывода.
- При необходимости установите права 755 на папку.

### У товаров нет изображений в карте
- Это ожидаемо для товаров, чья ресайз-миниатюра ещё не лежит в `image/cache`. Модуль по умолчанию не ресайзит во время генерации. Изображение добавится при следующем запуске после кэширования.

### Генерация медленная на огромном каталоге
- Используйте **Статический** режим через cron вместо динамического маршрута.
- Уменьшите **URL на один файл**, если отдельные файлы слишком большие.
