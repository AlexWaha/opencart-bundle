Instruction | Alexwaha.com - Age Verification for Opencart v2.3 -3.x

Installation Instructions

This module provides age verification (18+) popup that appears on first visit to restrict access for underage visitors

  • 1. Make sure you have installed the module aw_core_oc2.3-3.x.ocmod.zip This is an auxiliary function for all my modules
  • 2. Make sure you have a module installed that disables FTP uploads for extensions in Opencart 2.x, or that your FTP settings in the store config are correct – Link to module disabling FTP upload for Opencart v2.3 only (Not required for Opencart 3.x)
  • 3. Install the module files to your OpenCart installation:
    • Admin controller: admin/controller/extension/module/aw_age_verification.php
    • Admin model: admin/model/extension/module/aw_age_verification.php
    • Admin template: admin/view/template/extension/module/aw_age_verification.twig
    • Admin language files: admin/language/*/extension/module/aw_age_verification.php
    • Catalog controller: catalog/controller/extension/module/aw_age_verification.php
    • Catalog model: catalog/model/extension/module/aw_age_verification.php
    • Catalog template: catalog/view/theme/default/template/extension/module/aw_age_verification.twig
    • Catalog language files: catalog/language/*/extension/module/aw_age_verification.php
    • CSS styles: catalog/view/javascript/aw_age_verification.css
    • OCMOD file: system/aw_age_verification.ocmod.xml
  • 4. Refresh the modification cache
  • 5. Go to: Extensions Modules Enable the module – alexwaha.com - Age Verification When enabled, the module will automatically grant view/edit permissions

Self-check after installation:

  • Check for the module in the list of extensions
  • Refresh the modification cache
  • Test the age verification popup on the frontend (clear cookies first)
Configuration

  • 1. Go to Extensions Modules alexwaha.com - Age Verification

  • 2. Configure the module settings:
    • Status: Enable/Disable the age verification popup
    • Block Title: Set the popup title for each language (e.g., "Are you 18 years old?")
    • Description: Add popup description content for each language (supports rich text editor)
    • Cookie Storage Days: Set how long user response is remembered (1-365 days, default: 30)
    • Redirect URL: URL to redirect users who decline verification (optional)
  • 3. Save the configuration
  • 4. The popup will automatically appear on every page until user accepts it

The popup automatically integrates into all pages via OCMOD system modification

User response is stored in cookies and remembered for the configured period

How It Works

The age verification popup helps restrict site access to visitors under 18 years old.

  1. First Visit: Popup appears centered on screen with dark overlay
  2. User Choice: Visitor clicks "Yes" (I'm 18+) or "No" to confirm their age
  3. Accept: If "Yes", popup disappears and cookie is set for configured period
  4. Decline: If "No", user is redirected to specified URL (if configured)
  5. Return Visits: Popup won't show again until cookie expires

Features:
• Responsive design that works on all devices
• Smooth animations and professional styling
• Multilingual support
• Configurable cookie duration
• Optional redirect for declined verification

Important: This is a basic age verification system and should not be considered a foolproof age restriction method

Настройка

  • 1. Зайдите в раздел Дополнения Модули alexwaha.com - Подтверждение возраста

  • 2. Настройте параметры модуля:
    • Статус: Включить/Отключить всплывающее окно верификации возраста
    • Заголовок блока: Задайте заголовок всплывающего окна для каждого языка (например, "Вам исполнилось 18 лет?")
    • Описание: Добавьте текст описания для каждого языка (поддерживает визуальный редактор)
    • Срок хранения ответа (дни): Задайте, как долго помнить ответ пользователя (1-365 дней, по умолчанию: 30)
    • URL перенаправления: URL для перенаправления пользователей, отклонивших верификацию (необязательно)
  • 3. Сохраните конфигурацию
  • 4. Всплывающее окно будет автоматически появляться на всех страницах, пока пользователь не примет его

Всплывающее окно автоматически интегрируется во все страницы через систему модификаций OCMOD

Ответ пользователя сохраняется в куки и запоминается на настроенный период

Принцип работы

Всплывающее окно верификации возраста помогает ограничить доступ к сайту для посетителей младше 18 лет.

  1. Первое посещение: Появляется всплывающее окно по центру экрана с темным фоном
  2. Выбор пользователя: Посетитель нажимает "Да" (мне есть 18) или "Нет" для подтверждения возраста
  3. Принятие: Если "Да", окно исчезает и устанавливается кука на настроенный период
  4. Отклонение: Если "Нет", пользователь перенаправляется на указанный URL (если настроен)
  5. Повторные посещения: Окно больше не показывается до истечения срока куки

Особенности:
• Адаптивный дизайн, работающий на всех устройствах
• Плавная анимация и профессиональное оформление
• Поддержка многоязычности
• Настраиваемый срок хранения куки
• Опциональное перенаправление при отклонении верификации

Важно: Это базовая система верификации возраста и не должна рассматриваться как надежный метод ограничения доступа по возрасту
