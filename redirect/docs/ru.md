# Alexwaha.com - Redirect Manager

**Модуль для OpenCart v2.3 - 3.x**

---

## Описание

Redirect Manager управляет редиректами URL и отлавливает битые ссылки. Создан для миграций (смена домена, переезд с Shopify / PrestaShop / самописного движка / обычного HTML на OpenCart), когда старые URL могут быть в любом формате.

Два инструмента в одном модуле:

1. **Правила редиректов** - управляемый список `Исходный URL -> Целевой URL` с кодом редиректа (301 / 302 / 410) и статусом вкл/выкл. Правила срабатывают и для живых, и для мёртвых URL, обслуживаются из горячего кеша (без запроса в БД при попадании в кеш).
2. **Резолвер 404** - каждый URL, вернувший 404, логируется автоматически (с дедупликацией, счётчиком обращений и временем последнего обращения). Из лога администратор одним кликом создаёт редирект или отправляет выбранные URL на главную.

### Сопоставление

- **Точное**: `/old-page`.
- **Шаблон** с `*`: `/blog/*` -> `/news`. Источник с `*` автоматически считается шаблоном.
- **Query-строка**: по умолчанию сравнивается только путь. Включите «Учитывать query-строку» для правила, чтобы ловить старые не-ЧПУ URL вида `catalog.php?id=5` (порядок параметров не важен).
- Сопоставление **без учёта регистра** и **завершающего слеша**. Встроенная защита от петель не даёт создать правило, где цель совпадает с источником.

### Коды

- **301** Moved Permanently (по умолчанию)
- **302** Found (временный)
- **410** Gone - без редиректа, возвращает страницу «удалено» (цель не требуется)

## Инструкция по установке

1. **Установка основного модуля**

   Убедитесь в том, что у Вас установлен модуль **[aw_core_oc2.3-3.x.ocmod.zip](https://github.com/AlexWaha/opencart-bundle/blob/master/Core/dist/aw_core_oc2.3_3.x.ocmod.zip)**

   > Это вспомогательный функционал для работы всех моих модулей. **Критически необходимо**, без него работать модуль не будет

2. **Настройки FTP-загрузки (только для OpenCart 2.x)**

   Убедитесь, что загрузка дополнений по FTP разрешена (или настройки FTP корректны) на OpenCart 2.x. **(Для OpenCart 3.x не требуется)**

3. **Установите файлы модуля aw_redirect_oc2.3-3.x.ocmod.zip**
4. **Включение модуля**

   В разделе: **Расширения → Модули** → Включите модуль **alexwaha.com - Redirect Manager**

   > При включении модуль создаёт свои таблицы в БД, выдаёт права на просмотр и редактирование и регистрирует события витрины. Обновление кеша модификаций не требуется - модуль работает через систему событий OpenCart.

### Самопроверка после установки

- Откройте **Расширения → Модули → alexwaha.com - Redirect Manager** - откроется список правил
- Добавьте тестовый редирект, откройте исходный URL в новой вкладке - должно перенаправить на цель
- Откройте неизвестный URL, затем проверьте **Резолвер 404** - URL должен там появиться

## Использование

После включения модуль добавляет в левое меню админки пункт **Redirect Manager** с тремя разделами: **Правила редиректов**, **Резолвер 404** и **Настройки** (пункт скрыт, пока модуль выключен). Также можно открыть через **Расширения → Модули**.

### Правила редиректов

Главная страница - список всех правил (источник, цель, тип, код, счётчик срабатываний, статус). Панель фильтра ищет по источнику/цели, типу или статусу.

**Добавить / изменить правило:**

- **Исходный URL** - путь для сопоставления, напр. `/old-page` или `/blog/*`.
- **Учитывать query-строку** - включить query в сопоставление.
- **Целевой URL** - относительный путь (`/new-page`) или абсолютный URL (`https://...`). Не требуется для кода 410.
- **Код редиректа** - 301 / 302 / 410.
- **Магазин** - все магазины или конкретный.
- **Статус** - включить / отключить правило.

### Резолвер 404

URL, вернувшие 404, собираются здесь (с дедупликацией и счётчиком). Для каждой строки доступно:

- **Редирект** - открывает форму добавления с заполненным источником.
- **Редирект на главную** - массово создаёт 301-редиректы на `/` для выбранных строк.
- **Удалить** / **Очистить лог** - удаление записей.

### Настройки

- **Статус** - главный выключатель модуля на витрине.
- **Код по умолчанию** - код, выбранный при добавлении нового редиректа.
- **Логировать ошибки 404** - вкл/выкл автоматическое логирование 404.
- **Шаблоны исключений** - один шаблон на строку (можно с *); совпавшие 404-URL не логируются (по умолчанию отсекается шум ботов вроде `*.php`, `/wp-*`).
- **Импорт / Экспорт** - экспорт всех правил в CSV или массовый импорт из CSV (`source,target,status_code,match_query,store_id,status`).

## Для разработчиков

### Технический обзор

Модуль построен полностью на **системе событий OpenCart** - без модификаций OCMOD.

При установке регистрируются два события:

| Триггер | Действие | Назначение |
|---|---|---|
| `catalog/controller/*/before` | `extension/module/aw_redirect/redirect` | Сопоставляет запрос с активными правилами (один раз за запрос) и выполняет редирект |
| `catalog/controller/error/not_found/before` | `extension/module/aw_redirect/notFound` | Логирует отсутствующий URL (надёжнее чтения resolved route - работает и под SeoPro) |

### Производительность

Активные правила компилируются в одну запись **горячего кеша** (`aw_redirect.map`) через кеш OpenCart (файл или Redis - что использует магазин). При попадании в кеш - **ноль запросов в БД**: точный поиск по хешу за O(1) плюс проход по (немногим) wildcard-правилам в памяти. Кеш перестраивается автоматически после любого изменения. Для очень больших наборов (> 5000 точных правил) модуль переходит на индексированный точечный запрос по столбцу `source_hash` вместо кеширования всей карты.

### База данных

- `oc_aw_redirect` - правила (`source`, `source_hash`, `target`, `match_type`, `match_query`, `status_code`, `store_id`, `status`, `hits`).
- `oc_aw_redirect_404` - лог 404, дедупликация по `url_hash` со счётчиком `hits`.

### Расположение файлов

- **Событийный контроллер:** `catalog/controller/extension/module/aw_redirect.php`
- **Модель сопоставления / кеша:** `catalog/model/extension/module/aw_redirect.php`
- **Админ-контроллер / модель:** `admin/.../extension/module/aw_redirect.php`

> **Критическое требование:** Использование модуля возможно только вместе с системным модулем **alexwaha.com - Core**
