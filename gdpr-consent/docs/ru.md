# Alexwaha.com - GDPR Consent

**Модуль для OpenCart v2.3 - 3.x**

---

## Описание

Модуль добавляет баннер согласия на использование cookie (GDPR / ePrivacy) с полной поддержкой **Google Consent Mode v2**.

Баннер показывается в стиле Cookiebot - полосой внизу страницы (не центральным popup). Посетитель может принять все cookie, принять выбранные или отклонить необязательные. После выбора баннер скрывается, а вместо него появляется небольшой плавающий виджет, позволяющий открыть настройки повторно в любой момент.

Четыре категории согласия маппятся на семь сигналов Google Consent Mode v2, поэтому теги аналитики и рекламы получают корректное состояние `granted` / `denied`.

| Категория (UI) | Сигналы Google Consent Mode v2 |
|---|---|
| Необходимые (всегда включены) | `security_storage`, `functionality_storage` |
| Предпочтения | `personalization_storage` |
| Статистика | `analytics_storage` |
| Маркетинг | `ad_storage`, `ad_user_data`, `ad_personalization` |

По умолчанию все необязательные категории **выключены** (GDPR opt-in).

## Инструкция по установке

### Требования и этапы установки

1. **Установка основного модуля**

   Убедитесь в том, что у Вас установлен модуль **[aw_core_oc2.3-3.x.ocmod.zip](https://github.com/AlexWaha/opencart-bundle/blob/master/Core/dist/aw_core_oc2.3_3.x.ocmod.zip)**

   > Это вспомогательный функционал для работы всех моих модулей. **Критически необходимо**, без него работать модуль не будет

2. **Настройки FTP-загрузки (только для OpenCart 2.x)**

   Убедитесь в том, что у Вас установлен модуль отключающий FTP-загрузку дополнений в OpenCart 2.x или у Вас корректные настройки FTP в настройках магазина

   > [Ссылка на модуль отключающий FTP-загрузку только для OpenCart v2.3](https://www.opencart.com/index.php?route=extension/extension/info&extension_id=18892) **(Для OpenCart 3.x не требуется)**

3. **Установите файлы модуля aw_gdpr_consent_oc2.3-3.x.ocmod.zip**
4. **Включение модуля**

   В разделе: **Расширения → Модули** → Включите модуль **alexwaha.com - GDPR Consent**

   > При включении модуль автоматически выдаёт права на просмотр и редактирование и регистрирует события витрины. Обновление кеша модификаций не требуется - модуль работает исключительно через систему событий OpenCart.

### Самопроверка после установки

- Проверьте наличие модуля в списке расширений
- Включите и настройте модуль
- Откройте витрину в чистой сессии браузера (предварительно очистив cookie) - баннер согласия должен появиться внизу

## Настройка

### Параметры модуля

Перейдите в **Расширения → Модули → alexwaha.com - GDPR Consent**.

**Вкладка «Основные»:**

- **Статус:** включение / отключение баннера
- **Заголовок баннера:** заголовок на баннере, для каждого языка
- **Текст баннера:** поясняющий текст на баннере, для каждого языка (визуальный редактор)
- **Страница политики cookie:** информационная страница для ссылки «Подробнее»
- **Срок хранения cookie:** сколько дней запоминается выбор посетителя (1-730, по умолчанию 365)

**Вкладка «Внешний вид»:**

- **Тема:** светлая или тёмная цветовая схема баннера
- **Акцентный цвет:** цвет основной кнопки, ссылок и переключателей

Сохраните настройки. Баннер показывается на витрине, пока посетитель не сделает выбор.

### Сценарий взаимодействия

1. **Первый визит:** баннер появляется внизу страницы. Значения Google Consent Mode v2 по умолчанию выставлены в `denied` (кроме необходимых сигналов) **до** запуска любого тега аналитики.
2. **Принять все:** все категории разрешены, сигналы обновляются на `granted`, выбор сохраняется в cookie.
3. **Принять выбранные:** разрешаются только включённые посетителем категории.
4. **Отклонить:** включённой остаётся только необходимая категория.
5. **Повторные визиты / открытие:** после выбора баннер скрыт, а в левом нижнем углу показан плавающий виджет. Клик по нему снова открывает баннер для изменения решения.

### Возможности

- Нижний баннер в стиле Cookiebot с плавающим виджетом повторного открытия
- Google Consent Mode v2 (`gtag('consent', 'default' / 'update', ...)`)
- Четыре категории согласия, маппятся на семь сигналов Google
- Светлая / тёмная тема и настраиваемый акцентный цвет
- Мультиязычные заголовок и текст
- Работает на OpenCart 2.3 и 3.x

> **Примечание:** Модуль управляет только сигналами согласия. Сами теги Google Tag / GA4 / Ads подключаются отдельно (тема, другой модуль или раздел аналитики OpenCart).

## Для разработчиков

### Технический обзор

Модуль построен полностью на **системе событий OpenCart** - модификации OCMOD не используются.

При установке регистрируются два события:

| Триггер | Действие | Назначение |
|---|---|---|
| `catalog/view/common/header/after` | `extension/module/aw_gdpr_consent/head` | Вставляет состояние Consent Mode v2 по умолчанию + ассеты в начало `<head>` |
| `catalog/view/common/footer/after` | `extension/module/aw_gdpr_consent/footer` | Вставляет баннер + виджет перед `</body>` |

Оба обработчика изменяют строку результата рендера, поэтому работают одинаково на OpenCart 2.3 и 3.x.

### Расположение файлов

- **Шаблон витрины:** `catalog/view/theme/default/template/extension/module/aw_gdpr_consent.twig`
- **Шаблон head:** `catalog/view/theme/default/template/extension/module/aw_gdpr_consent_head.twig`
- **CSS-стили:** `catalog/view/javascript/aw_gdpr_consent.css`
- **JavaScript:** `catalog/view/javascript/aw_gdpr_consent.js`

### Ключевые компоненты

- **Внедрение через события** - без модификаций шаблонов и кеша модификаций
- **Consent Mode v2** - состояние по умолчанию восстанавливается из cookie при повторных визитах
- **Cookie по сигналам** - `ad_storage`, `analytics_storage` и т. д., читаются head-скриптом
- **Версионирование ассетов** - cache-busting `?v=filemtime` для CSS / JS

> **Критическое требование:** Использование модуля возможно только вместе с системным модулем **alexwaha.com - Core**
