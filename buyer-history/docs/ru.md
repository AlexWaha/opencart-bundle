## Информация о модуле

**Buyer History** - модуль для OpenCart, который добавляет в админку:

- **Колонку «История заказов»** в стандартный список заказов (`Sales > Orders`) - цветной бейдж с количеством всех заказов клиента, разбивкой по статусам и суммами в выпадающем меню.
- **Колонку «Дубликаты»** - номера других заказов того же клиента в настраиваемом окне времени.
- **Отдельную страницу «История покупателей»** в меню `Отчёты` - список уникальных клиентов с агрегатами (всего заказов, средний чек, первый/последний заказ), фильтрами по уровню/email/наличию дубликатов и разворачиваемой строкой со списком заказов и составом каждого заказа (товары + опции + количество + цена).

Менеджер сразу видит постоянных клиентов и не звонит лишний раз тем, кто уже многократно у вас заказывал.

## Установка

1. Установите модуль `alexwaha.com - Core` (требуется как зависимость).
2. Зайдите в `Дополнения → Установка дополнений` и загрузите `aw_buyer_history_oc2.3-3.x.ocmod.zip`.
3. Зайдите в `Дополнения → Модификаторы`, нажмите кнопку «Обновить» (значок refresh). На OpenCart 2.3 это применяет OCMOD из комплекта (`system/aw_buyer_history.ocmod.xml`), который вставляет разметку колонок в шаблон списка заказов. На 3.x OCMOD не задействован - колонки вставляются на лету через event system.
4. Зайдите в `Дополнения → Дополнения → Модули`, найдите `alexwaha.com - Buyer History` и нажмите зелёный `+` (Установить). При установке модуль автоматически:
   - Регистрирует два OpenCart-события: `admin/view/sale/order_list/before` и `admin/view/common/column_left/before`.
   - Создаёт пункт меню в `Отчёты → История покупателей`.
   - Назначает permission `access`/`modify` для текущей user-group.
5. Откройте `Дополнения → Модули → alexwaha.com - Buyer History → Редактировать`, включите статус и сохраните.

После установки в `Sales > Orders` появятся новые колонки, а в меню `Отчёты` - пункт «История покупателей».

## Колонки в Sales > Orders

После включения модуля в `Sales > Orders` рядом со столбцом «Покупатель» появляются:

### История заказов

Цветной бейдж с количеством заказов клиента. По клику открывается выпадающее меню с разбивкой по статусам, количеством и суммами в базовой валюте магазина:

```
По статусам
В обработке   4   7 100 грн
Доставлено    4   7 300 грн
Ожидание      4   7 500 грн
Всего        12  21 900 грн
```

Цвета бейджа настраиваются по уровням (Низкий / Средний / Высокий) с порогами по числу заказов.

### Дубликаты

Номера других заказов того же клиента, попавших в окно времени (от 1 часа до 7 дней - настраивается). При превышении лимита отображения появляется кнопка `+N` с выпадающим меню всех остальных номеров (со скроллом для сотен дубликатов).

### Идентификация клиента

- Зарегистрированные: по `customer_id`
- Гости: по `email + нормализованный телефон` (если включено в настройках)

## Страница «История покупателей»

Доступна через `Отчёты → История покупателей`. Маршрут: `index.php?route=extension/module/aw_buyer_history/report`.

### Колонки

| Колонка | Описание |
|---------|----------|
| Покупатель | Имя, email, телефон |
| Заказов | Количество с цветным tier-бейджем |
| Сумма всего | Lifetime spend в базовой валюте |
| Средний чек | AVG(total) |
| Первый заказ | Дата первого заказа |
| Последний заказ | Дата последнего заказа |
| Дубли | Иконка ⚠ если у клиента есть дубликаты в окне |

### Фильтры

- **Поиск** - partial match по email, телефону, имени.
- **Уровень** - Все / Начальный / Средний / Высокий.
- **Только с дубликатами** - оставляет только клиентов с дубликатами в окне.

### Разворачиваемая строка

Клик по chevron → AJAX-загрузка списка всех заказов клиента. Под каждой строкой заказа выводится:

- Номер заказа
- Цветной бейдж статуса (цвета настраиваются)
- Количество позиций
- Сумма (жирная, по центру)
- Дата (по центру)
- Кнопка перехода в `sale/order/info`
- Список товаров: имя со ссылкой на `catalog/product/edit`, артикул, выбранные опции (как chips), количество × цена

Строки клиентов с дубликатами подсвечены лёгким красным фоном.

## Настройки

### Общие

- **Статус** - главный гейт. При выключении модуля колонки и пункт меню исчезают.
- **Учитывать гостей по email + телефону** - связывать заказы гостей по контактам.
- **Учитывать статусы заказов** - чекбоксы по всем статусам OpenCart. Снятые статусы не считаются в счётчике / разбивке.

### Пороги и цвета

- **Порог среднего уровня** (по умолчанию 3 заказа)
- **Порог высокого уровня** (по умолчанию 10 заказов)
- **Цвета бейджей** для каждого уровня (Low / Mid / High) - фон + текст. Coloris color picker. Live preview обновляется при вводе hex.

### Цвета статусов

Для каждого статуса заказа можно задать свой цвет. На странице «История покупателей» в развёрнутой строке клиента статусы заказов будут подсвечены этим цветом. Имя статуса слева от поля ввода также превращается в цветной label при вводе hex (live preview).

### Дубликаты

- **Включить детект дубликатов** - switch.
- **Временное окно** - пресеты `1ч / 3ч / 6ч / 12ч / 24ч / 48ч / 72ч / 7 дней` или своё значение в минутах/часах/днях.
- **Минимум заказов в окне для пометки** - по умолчанию 2 (есть хотя бы один дубликат).
- **Максимум номеров в ячейке** - лишние сворачиваются в кнопку `+N` с dropdown-списком (со скроллом для большого количества).
- **Цвет бейджа дубликата** - фон + текст (Coloris).
- **Открывать ссылки** - `В том же окне` / `В новой вкладке`.

### Отображение

- **Колонка «История заказов»** - switch.
- **Колонка «Дубликаты»** - switch.

### Импорт / Экспорт

- **Экспортировать** - скачивает все настройки модуля как `aw_buyer_history_settings_<date>.json`.
- **Импортировать** - выбор JSON файла → перезаписывает текущие настройки.

## Архитектура

Способ вывода колонок зависит от версии OpenCart. Пункт меню в «Отчётах» и все данные строк всегда приходят через event system; различается только разметка колонок в списке заказов между 2.3 и 3.x.

### Колонки в списке заказов: разделение по платформам

| Платформа | Разметка колонок | Данные строк |
|-----------|------------------|--------------|
| OpenCart 3.x | Event патчит Twig-шаблон в памяти (core-файлы не меняются) | Event |
| OpenCart 2.3 | OCMOD из комплекта патчит `order_list.tpl` + `header.tpl` | Event |

На **3.x** событие `admin/view/sale/order_list/before` читает `admin/view/template/sale/order_list.twig`, выполняет `str_replace` для вставки фрагментов `<th>`/`<td>` из `injectors/order_list_headers.twig` и `injectors/order_list_cells.twig` и подкладывает результат в `$code` параметр view-метода - Twig движок (`system/library/template/twig.php:14-44`) рендерит пропатченную разметку. Ни один файл на диске не меняется.

На **2.3** legacy-шаблон - `order_list.tpl`, а патч `$code` в памяти намеренно пропускается (контроллер закрывает его проверкой `isLegacy()`). Разметку колонок даёт OCMOD из комплекта `system/aw_buyer_history.ocmod.xml`, который добавляет блоки шапки/ячеек в `order_list.tpl` и ссылку на стиль в `header.tpl`. Event при этом всё равно отрабатывает - он лишь заполняет в каждой строке `$data['orders']` поле `aw_history` (счётчик, бейдж, дубликаты).

### Event: `admin/view/common/column_left/before`

Добавляет в массив `$data['menus']` пункт «История покупателей» внутри Reports submenu. Если у пользователя нет permission на Reports, добавляет отдельный root-пункт.

### Индексы в БД

Чтобы агрегации на каждой странице оставались быстрыми на больших таблицах заказов, при установке модуль добавляет два индекса в `oc_order`: `aw_bh_email` (`email`) и `aw_bh_date_added` (`date_added`). Они переводят выборки истории и дубликатов на доступ по индексу вместо полного скана таблицы. Оба удаляются при деинсталляции.

### Очистка

После удаления модуля и нажатия `Modifications → Refresh` правки OCMOD откатываются, добавленные индексы удаляются, а зарегистрированные события снимаются. Core-шаблоны OpenCart остаются нетронутыми.
