## Информация о модуле

**AW Microdata** - это профессиональный модуль для OpenCart, который автоматически генерирует структурированные данные Schema.org (JSON-LD) для всех страниц вашего интернет-магазина. Модуль использует встроенную систему событий (Events) OpenCart для внедрения разметки, что делает его полностью независимым от шаблона и не требует OCMOD-модификаций.

Структурированные данные помогают поисковым системам лучше понимать содержимое вашего сайта, что может привести к появлению расширенных сниппетов в результатах поиска (звёздные рейтинги, цены, наличие, FAQ-аккордеоны, хлебные крошки и др.).

### Основные возможности:

- **25+ типов Schema.org** - Product, Organization, FAQPage, ContactPage, BreadcrumbList, WebSite, BlogPosting и многие другие
- **Независимость от шаблона** - Использует систему событий OpenCart, OCMOD-модификации не нужны
- **Встроенная диагностика** - Проверка регистрации событий, обнаружение дублирующей разметки, проверка заполненности конфигурации
- **OpenGraph и Twitter Cards** - Полная генерация OG и Twitter мета-тегов
- **Автоопределение VideoObject** - Автоматически находит ссылки на YouTube/Vimeo в описаниях товаров и статей блога
- **Извлечение ImageObject** - Извлекает изображения из описаний и генерирует ImageObject-разметку
- **ItemList / Карусель** - Генерирует ItemList-разметку для страниц категорий (карусель товаров в Google)
- **Сопутствующие товары** - Включает связанные/рекомендуемые товары как `isRelatedTo` в Product-разметку
- **Маппинг атрибутов** - Привязка атрибутов OpenCart к свойствам Schema.org (цвет, материал, размер, пол)
- **Накрутка рейтинга и отзывов** - Настраиваемое увеличение рейтинга и количества отзывов для SEO
- **Конкурентные sameAs** - Серый SEO: ассоциация брендовых запросов конкурентов с вашим сайтом
- **Произвольный JSON-LD** - Вставка любого пользовательского JSON-LD блока на каждую страницу
- **Импорт/Экспорт настроек** - Резервное копирование и восстановление конфигурации модуля в JSON
- **Без внешних API** - Всё работает локально, без обращений к сторонним сервисам
- **Мультиязычность** - Английский, русский, украинский

## Инструкция по установке

### Шаг 1: Предварительные требования

1. Убедитесь, что у вас установлен модуль **[aw_core_oc2.3-3.x.ocmod.zip](https://github.com/AlexWaha/opencart-bundle/blob/master/Core/dist/aw_core_oc2.3_3.x.ocmod.zip)**

   > **Важно:** Это вспомогательный функционал, необходимый для работы всех модулей AlexWaha

2. Для OpenCart 2.3.x убедитесь, что у вас установлен модуль, отключающий FTP-загрузку дополнений, или настроены корректные параметры FTP в настройках магазина

   > **Примечание:** Для OpenCart 3.x это не требуется

   [Ссылка на модуль отключения FTP-загрузки (только для OpenCart v2.3)](https://www.opencart.com/index.php?route=extension/extension/info&extension_id=18892)

### Шаг 2: Установка модуля

1. Установите архив **aw_microdata_oc2.3-3.x.ocmod.zip** через админ-панель сайта

   - Перейдите в **Расширения -> Установщик**
   - Загрузите архив модуля
   - Дождитесь завершения установки

2. Обновите кеш модификаций

   - Перейдите в **Расширения -> Модификаторы**
   - Нажмите кнопку **Обновить**

3. В разделе **Расширения -> Модули** найдите и включите модуль:

   **alexwaha.com - Microdata**

   > **Важно:** При включении модуль автоматически зарегистрирует 12 обработчиков событий и настроит права доступа. Таблицы в БД не создаются - вся конфигурация хранится в общей таблице `aw_module_config`, управляемой AwCore.

### Шаг 3: Самопроверка после установки

Проверьте корректность установки:

- Модуль присутствует в списке расширений
- Страница настроек модуля открывается без ошибок
- Вкладка "Диагностика" показывает все 12 событий со статусом "OK"
- Предупреждения о дублирующей разметке отсутствуют

### Вкладка "Диагностика"

Вкладка "Диагностика" выполняет комплексную проверку состояния модуля. Проверка запускается автоматически при открытии и отображает результаты в трёх категориях.

#### Статус событий

Проверяет, все ли 12 обработчиков событий корректно зарегистрированы в системе событий OpenCart. Каждое событие отображается отдельно с его статусом:

| Триггер события | Назначение | Ожидаемый статус |
|---|---|---|
| `catalog/view/common/header/after` | Внедрение OG мета-тегов в `<head>` | OK |
| `catalog/view/common/footer/after` | Внедрение JSON-LD организации перед `</body>` | OK |
| `catalog/view/common/home/after` | Разметка WebSite + CollectionPage на главной странице | OK |
| `catalog/view/product/product/after` | Разметка Product + BreadcrumbList | OK |
| `catalog/view/product/category/after` | Разметка CollectionPage + BreadcrumbList | OK |
| `catalog/view/product/search/after` | Разметка SearchResultsPage + BreadcrumbList | OK |
| `catalog/view/product/manufacturer_info/after` | Разметка CollectionPage + Brand + BreadcrumbList | OK |
| `catalog/view/product/special/after` | Разметка CollectionPage + BreadcrumbList | OK |
| `catalog/view/information/information/after` | Разметка Article для информационных страниц | OK |
| `catalog/view/information/contact/after` | Разметка ContactPage | OK |
| `catalog/view/blog/article/after` | Разметка BlogPosting для статей блога | OK |
| `catalog/view/blog/category/after` | Разметка Blog для категорий блога | OK |

Если какое-либо событие показывает статус "ОТСУТСТВУЕТ", переустановите модуль (отключите и включите заново в разделе Расширения -> Модули).

#### Обнаружение дублирующей разметки

Сканирует все `.twig`-файлы шаблонов в директории `catalog/view/theme/` на наличие хардкодных блоков `<script type="application/ld+json">`, не сгенерированных AW Microdata. Если такие найдены, они могут конфликтовать с выводом модуля и должны быть удалены из шаблонов.

#### Полнота конфигурации

Проверяет заполненность рекомендуемых полей:

- **Логотип** (вкладка "Организация") - необходим для разметки Organization и Publisher
- **Телефоны** (вкладка "Организация") - необходимы для `telephone` и `contactPoint`
- **Адрес** (вкладка "Организация") - необходим для `PostalAddress`
- **Email** (вкладка "Организация") - необходим для разметки Organization

Незаполненные поля перечисляются со ссылкой на соответствующую вкладку настроек.

#### Ссылки на Google Rich Results Test

Предоставляет готовые ссылки на инструмент Google Rich Results Test для ключевых страниц вашего магазина:

- Главная страница
- Первый активный товар
- Первая активная категория
- Страница контактов

Нажмите на ссылку, чтобы проверить структурированные данные напрямую через Google.

### Основные настройки

#### Статус модуля

- **Включён** - Модуль активен и внедряет структурированные данные на все страницы
- **Выключен** - Модуль полностью отключён, разметка не генерируется

#### Формат вывода

На данный момент поддерживается только **JSON-LD** (рекомендован Google). JSON-LD внедряется в виде блоков `<script type="application/ld+json">`.

#### Название сайта

Название вашего сайта, которое будет использоваться в разметке WebSite. Настраивается для каждого языка. Если не заполнено, берётся название магазина из настроек OpenCart.

#### Альтернативное название

Альтернативное название сайта (например, сокращение или популярное название). Отображается как `alternateName` в разметке WebSite.

#### URL поиска по сайту

Шаблон URL для функции поиска. Используется для генерации `SearchAction` в разметке WebSite, что позволяет активировать поисковую строку в результатах Google.

**По умолчанию:** `{shop_url}/index.php?route=product/search&search={search_term_string}`

> **Совет:** Если у вас нестандартный URL поиска, укажите его здесь. Плейсхолдер `{search_term_string}` обязателен для Google.

#### Изображение по умолчанию

Резервное изображение, используемое когда у страницы нет собственного изображения. Применяется для OpenGraph-тегов и разметки Schema.org.

### Организация

Вкладка "Организация" настраивает структурированные данные о вашем бизнесе. Эти данные отображаются на каждой странице в виде разметки Organization (или подтипа).

#### Тип магазина

Выберите тип Schema.org, наиболее точно описывающий ваш бизнес. Доступные варианты:

- `Store` (по умолчанию)
- `OnlineStore`
- `LocalBusiness`
- `LiquorStore`
- `ClothingStore`
- `ElectronicsStore`
- `FurnitureStore`
- `GardenStore`
- `GroceryStore`
- `HardwareStore`
- `HobbyShop`
- `HomeGoodsStore`
- `JewelryStore`
- `MusicStore`
- `OfficeEquipmentStore`
- `PetStore`
- `ShoeStore`
- `SportingGoodsStore`
- `TireShop`
- `ToyStore`
- `WholesaleStore`
- `AutoPartsStore`
- `BikeStore`
- `BookStore`
- `ComputerStore`
- `ConvenienceStore`
- `DepartmentStore`
- `Florist`
- `Optician`
- `OutletStore`
- И другие (30+ вариантов)

> **Рекомендация:** Выберите наиболее конкретный тип, соответствующий вашему бизнесу. `OnlineStore` - хороший вариант по умолчанию для интернет-магазина.

#### Юридическое название

Официальное юридическое название вашей организации. Настраивается для каждого языка. Отображается как `legalName` в разметке. Если не заполнено, используется название магазина.

#### Email

Контактный email организации. Отображается как `email` в разметке Organization.

#### Телефоны

Один или несколько телефонных номеров. Нажмите "Добавить" для добавления дополнительных номеров. Отображаются как `telephone` в разметке. Первый номер также используется для свойства `contactPoint`.

#### Социальные сети

URL-адреса ваших профилей в социальных сетях (Facebook, Instagram, Twitter/X, YouTube, Telegram и т.д.). Каждый URL добавляется как значение `sameAs` в разметке Organization. Нажмите "Добавить" для добавления новых ссылок.

#### Адрес

Физический адрес вашего бизнеса, настраиваемый для каждого языка:

- **Улица** - Адрес улицы (отображается как `streetAddress`)
- **Город** - Населённый пункт (отображается как `addressLocality`)
- **Область** - Регион/область (отображается как `addressRegion`)
- **Почтовый индекс** - Индекс (отображается как `postalCode`)
- **Страна** - Код страны, например UA, US, DE (отображается как `addressCountry`). Если не заполнено, берётся из настроек магазина OpenCart.

#### Геокоординаты

Широта и долгота расположения вашего бизнеса. Отображаются как `GeoCoordinates` в разметке.

**Формат:** Десятичные градусы (например, `50.4501` для широты, `30.5234` для долготы)

> **Совет:** Координаты можно найти в Google Maps - щёлкните правой кнопкой мыши по нужному месту и скопируйте координаты.

#### Логотип

URL или путь к изображению логотипа организации. Если не заполнено, используется логотип магазина из настроек OpenCart. Отображается как `logo` и `image` в разметке Organization.

#### Время работы

Расписание для каждого дня недели (с понедельника по воскресенье). Для каждого дня настраивается:

- **Открытие** - Время открытия (например, `09:00`)
- **Закрытие** - Время закрытия (например, `20:00`)
- **Выходной** - Отметьте, чтобы обозначить нерабочий день

Отображается как `openingHoursSpecification` в разметке.

#### Код валюты

Валюта, принимаемая вашим бизнесом (например, `UAH`, `USD`, `EUR`). По умолчанию берётся из настроек магазина. Отображается как `currenciesAccepted`.

#### Ценовой диапазон

Индикатор ценового диапазона в условных обозначениях:

- `$` - Бюджетный
- `$$` - Средний
- `$$$` - Дорогой
- `$$$$` - Премиум

Отображается как `priceRange` в разметке Organization.

#### Способы оплаты

Принимаемые способы оплаты (свободный текст). Отображается как `paymentAccepted` в разметке.

**Пример:** `Наличные, Банковская карта, PayPal, Банковский перевод`

#### Зоны доставки

Территории, на которые осуществляется доставка (свободный текст). Отображается как `areaServed` в разметке.

**Пример:** `Киев, Харьков, Одесса, Вся Украина`

### Товары

Вкладка "Товары" настраивает разметку Schema.org `Product` для карточек товаров.

#### Разметка товаров

Переключатель включения/выключения генерации Product-разметки на страницах товаров. По умолчанию включён.

#### Рейтинг товаров

При включении добавляет `aggregateRating` в Product-разметку на основе отзывов о товаре. По умолчанию включён.

#### Источник отзывов

Определяет, какие отзывы используются для Product-разметки:

- **Только отзывы товара** - Только отзывы конкретного товара
- **Только отзывы магазина** - Общие отзывы магазина (из модуля AW Reviews)
- **Оба** - Комбинация отзывов товара и магазина

> **Рекомендация:** Используйте "Только отзывы товара" для наиболее точного представления. Варианты "Отзывы магазина" или "Оба" полезны, если у отдельных товаров мало отзывов.

#### Мин. количество отзывов

Минимальное количество отзывов, необходимое для включения `aggregateRating` в разметку. По умолчанию: `1`.

#### Мин. рейтинг

Минимальный средний рейтинг, необходимый для включения `aggregateRating`. По умолчанию: `1`.

#### Цена за единицу

При включении добавляет `UnitPriceSpecification` к предложению товара. Полезно для товаров, продаваемых по весу, объёму или другим единицам.

#### Код единицы (UN/CEFACT)

Код единицы измерения по стандарту UN/CEFACT:

- `LTR` - Литр
- `KGM` - Килограмм
- `MTR` - Метр
- `CMT` - Сантиметр
- `GRM` - Грамм

#### Референсное количество

Базовое количество для цены за единицу. По умолчанию: `1`.

#### Состояние товара

Состояние товаров в вашем магазине:

- `NewCondition` (по умолчанию) - Новые товары
- `UsedCondition` - Бывшие в употреблении
- `RefurbishedCondition` - Восстановленные
- `DamagedCondition` - С повреждениями

#### Источник бренда

Откуда брать название бренда для товаров:

- **Производитель** (по умолчанию) - Используется название производителя товара
- **Название магазина** - Используется название магазина как бренд

#### Маппинг наличия

Привязка статусов наличия OpenCart к значениям доступности Schema.org. Для каждого статуса наличия в вашем магазине можно назначить соответствующее значение Schema.org:

- `https://schema.org/InStock`
- `https://schema.org/OutOfStock`
- `https://schema.org/PreOrder`
- `https://schema.org/BackOrder`
- `https://schema.org/SoldOut`
- `https://schema.org/LimitedAvailability`

#### Срок доставки

При включении добавляет `shippingDetails` с `deliveryTime` к предложению товара.

- **Мин. дней** - Минимальный срок доставки в днях (по умолчанию: `1`)
- **Макс. дней** - Максимальный срок доставки в днях (по умолчанию: `3`)

#### Политика возврата

При включении добавляет `hasMerchantReturnPolicy` к предложению товара.

- **Дней возврата** - Количество дней на возврат (по умолчанию: `14`)
- **Тип возврата** - Категория политики возврата Schema.org:
  - `MerchantReturnFiniteReturnWindow` (по умолчанию) - Возврат принимается в определённый срок
  - `MerchantReturnNotPermitted` - Возврат не принимается
  - `MerchantReturnUnlimitedWindow` - Неограниченный срок возврата

#### Детали доставки

При включении добавляет информацию о доставке в разметку предложения товара.

#### Включить вес

При включении добавляет вес товара в Product-разметку как `QuantitativeValue`.

#### Включить размеры

При включении добавляет габариты товара (длина, ширина, высота) в Product-разметку.

#### Все изображения

При включении в Product-разметку включаются все изображения товара (не только основное). Включено по умолчанию.

#### Включить атрибуты

При включении все атрибуты товара добавляются как `additionalProperty` (PropertyValue) в Product-разметку.

#### Маппинг атрибутов

Привязка конкретных атрибутов OpenCart к свойствам Schema.org:

- **Атрибут для цвета** - Выберите атрибут, обозначающий цвет товара (отображается как `color`)
- **Атрибут для материала** - Выберите атрибут, обозначающий материал (отображается как `material`)
- **Атрибут для размера** - Выберите атрибут, обозначающий размер (отображается как `size`)
- **Целевой пол** - Выберите целевую аудиторию по полу:
  - Мужской
  - Женский
  - Унисекс

> **Совет:** Оставьте "Не привязан", если у ваших товаров нет этих атрибутов.

#### Сопутствующие товары (isRelatedTo)

При включении добавляет связанные/рекомендуемые товары со страницы товара как `isRelatedTo` в Product-разметку.

#### VideoObject из описания

При включении автоматически находит ссылки на YouTube и Vimeo в описаниях товаров и генерирует `VideoObject`-разметку для каждого видео. Работает без обращений к внешним API - миниатюры генерируются на основе ID видео.

#### ImageObject из описания

При включении извлекает теги `<img>` из описаний товаров и генерирует `ImageObject`-разметку для каждого изображения.

### Листинги (Категории и списки)

Вкладка "Листинги" настраивает структурированные данные для всех страниц с товарными листингами: категории, поиск, производители, акции, лендинги и главная страница.

Все листинги используют общую структуру: `CollectionPage` (или настраиваемый тип) с `AggregateOffer` (ценовой диапазон), опциональным `AggregateRating` (композитный рейтинг), опциональной информацией о доставке и возврате, и опциональной каруселью `ItemList`.

#### Разметка категорий

Переключатель включения/выключения генерации разметки для страниц категорий. По умолчанию включён.

#### Тип схемы

Тип Schema.org для страниц категорий:

- `CollectionPage` (по умолчанию) - Страница-коллекция
- `ItemList` - Список элементов
- `OfferCatalog` - Каталог предложений

> **Рекомендация:** `CollectionPage` - наиболее семантически корректный тип для страниц категорий.

#### Разметка поиска

Переключатель включения/выключения генерации `SearchResultsPage`-разметки на страницах результатов поиска. Включает `AggregateOffer` с ценовым диапазоном найденных товаров. По умолчанию включён.

#### Разметка производителя

Переключатель включения/выключения генерации `CollectionPage`-разметки на страницах товаров производителя. Включает `AggregateOffer` с ценовым диапазоном и свойство `Brand` с названием производителя. По умолчанию включён.

#### Разметка акций

Переключатель включения/выключения генерации `CollectionPage`-разметки на странице акций. Использует **акционные/скидочные цены** (не обычные) для ценового диапазона `AggregateOffer`. По умолчанию включён.

#### Товары на главной

При включении генерирует `CollectionPage`-разметку на главной странице с `AggregateOffer`, содержащим ценовой диапазон по всем активным товарам магазина. По умолчанию выключён.

> **Примечание:** Это дополнение к `WebSite`-разметке, уже генерируемой на главной.

#### Ценовой диапазон

При включении запрашивает из базы данных минимальную и максимальную цену товаров и генерирует `AggregateOffer` с `lowPrice`, `highPrice` и `offerCount`. Применяется ко всем типам листинговых страниц.

#### Количество товаров

При включении отображает общее количество активных товаров в листинге.

#### Композитный рейтинг на листингах

При включении рассчитывает **композитный агрегированный рейтинг**, объединяя отзывы на товары (из таблицы `review`) и отзывы на магазин (из таблицы `aw_review`), и добавляет `aggregateRating` в разметку листинга. Это даёт более полный рейтинг, даже если на отдельных страницах мало отзывов.

> **Примечание:** Композитный рейтинг используется на всех листинговых страницах (категории, поиск, производитель, акции, лендинги, главная). Настройки накрутки рейтинга из вкладки "Продвинутые" также применяются.

#### Доставка на листингах

При включении добавляет `shippingDetails` с `deliveryTime` в `AggregateOffer` на всех листинговых страницах. Использует те же значения **Мин. дней** и **Макс. дней**, настроенные во вкладке "Товары".

#### Возврат на листингах

При включении добавляет `hasMerchantReturnPolicy` в `AggregateOffer` на всех листинговых страницах. Использует те же значения **Дней возврата** и **Тип возврата**, настроенные во вкладке "Товары".

#### Карусель товаров (ItemList)

При включении генерирует `ItemList`-разметку для товаров, отображаемых в листинге. Google может показать это как карусель товаров в результатах поиска.

Каждый товар указывается как `ListItem` с позицией, URL и названием.

#### Разметка лендингов

При включении генерирует `CollectionPage`-разметку для лендингов (если установлен модуль AW Landing Pages). Включает ценовой диапазон товаров на лендинге.

#### Зона обслуживания

При включении использует заголовок лендинга как значение `areaServed`. Полезно для лендингов, привязанных к локациям (например, "Доставка в Киеве").

### Контентные страницы

Вкладка "Контентные страницы" настраивает структурированные данные для различных типов контентных страниц: информационных, блога, FAQ, отзывов, калькулятора и контактов.

#### Разметка инфо-страниц

Переключатель включения/выключения Article-разметки для информационных страниц OpenCart. По умолчанию включён.

#### Тип схемы (информационные страницы)

Тип Schema.org для информационных страниц:

- `Article` (по умолчанию) - Общая статья
- `NewsArticle` - Новостная статья
- `TechArticle` - Техническая статья
- `ScholarlyArticle` - Научная статья

#### Имя автора

Имя автора для статей. Если не заполнено, используется название магазина с типом Organization.

#### URL автора

URL автора. Используется вместе с именем автора для свойства `author`.

#### Разметка блога

При включении генерирует `BlogPosting` (или выбранный тип) разметку для страниц статей блога (требуется модуль блога).

#### Тип блога

Тип Schema.org для статей блога:

- `BlogPosting` (по умолчанию) - Стандартная публикация в блоге
- `Article` - Общая статья
- `NewsArticle` - Новостная статья
- `SocialMediaPosting` - Публикация в соцсетях

#### Количество слов

При включении подсчитывает и добавляет свойство `wordCount` в разметку статей блога.

#### Разметка FAQ

При включении генерирует `FAQPage`-разметку на страницах с данными FAQ (требуется модуль AW FAQ). Вопросы и ответы извлекаются из таблиц `aw_faq` и `aw_faq_description`.

Генерируемая разметка соответствует требованиям Google для расширенных результатов FAQ:

```json
{
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Какова ваша политика возврата?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Мы принимаем возврат в течение 14 дней."
      }
    }
  ]
}
```

#### Разметка страницы отзывов

При включении генерирует Organization-разметку с `aggregateRating` и отдельными записями `review` на странице отзывов. Данные берутся из отзывов магазина (модуль AW Reviews).

#### Разметка калькулятора

При включении генерирует `WebApplication`-разметку для страниц калькуляторов (например, AW Moonshine Calculator).

- **Категория приложения:** `UtilitiesApplication` (по умолчанию)
- **Цена:** Бесплатно (Offer с ценой 0)

#### HowTo для калькулятора

При включении дополнительно генерирует `HowTo`-разметку вместе с WebApplication - пошаговую инструкцию по использованию калькулятора.

#### Разметка контактов

При включении генерирует `ContactPage`-разметку на странице контактов с данными организации `LocalBusiness` в качестве `mainEntity`. Включает адрес, телефон, часы работы и другие данные организации.

### OpenGraph

Вкладка "OpenGraph" настраивает мета-теги Open Graph и Twitter Card для корректного отображения при публикации в социальных сетях.

#### Включить OpenGraph

Переключатель включения/выключения генерации OpenGraph мета-тегов. При включении в `<head>` добавляются следующие теги:

- `og:title` - Заголовок страницы
- `og:description` - Описание страницы (из мета-описания или контента)
- `og:url` - Канонический URL страницы
- `og:site_name` - Название магазина
- `og:locale` - Локаль текущего языка
- `og:image` - Изображение страницы (изображение товара, статьи или логотип)
- `og:type` - Тип контента (автоопределение: `product` для товаров, `article` для статей, настраиваемое значение для остальных страниц)

Для страниц товаров генерируются дополнительные теги:

- `product:price:amount` - Цена товара
- `product:price:currency` - Код валюты

#### Тип OG по умолчанию

Тип Open Graph по умолчанию для страниц без конкретного типа:

- `website` (по умолчанию) - Общий сайт
- `article` - Статья
- `product` - Товар

#### Facebook App ID

ID вашего приложения Facebook для интеграции с Facebook Insights. Необязательно.

#### Facebook Pages

ID вашей страницы (страниц) Facebook. Необязательно.

#### Тип Twitter Card

Тип генерируемой Twitter Card:

- `summary_large_image` (по умолчанию) - Карточка с большим изображением
- `summary` - Стандартная карточка

#### Twitter Username

Ваше имя пользователя в Twitter/X (например, `@yourusername`). Используется для мета-тега `twitter:site`.

#### OG изображение по умолчанию

Резервное изображение для OpenGraph, когда у страницы нет собственного изображения. Если не указано, используется логотип магазина.

### Продвинутые / Трюки

> **Внимание:** Эти настройки могут повлиять на ваше SEO как положительно, так и отрицательно. Используйте с осторожностью.

#### Глобальный AggregateRating

При включении добавляет `AggregateRating` в разметку Organization на всех страницах, а не только на товарных. Это означает, что агрегированный рейтинг вашего магазина будет отображаться на всём сайте.

> **Применение:** Полезно, если вы хотите, чтобы звёздные рейтинги отображались в Google для главной страницы или любой другой страницы.

#### Накрутка количества отзывов

Статическое число, прибавляемое к реальному количеству отзывов в Schema-разметке. Например, если у вас 5 реальных отзывов и вы указали `10`, Google увидит 15 отзывов.

Установите `0` для отключения.

> **Внимание:** Это серая SEO-техника. Google может наказать сайты, искажающие данные об отзывах.

#### Накрутка рейтинга

Десятичное значение (0.0-5.0), прибавляемое к реальному среднему рейтингу. Например, если ваш реальный средний рейтинг 4.5 и вы указали `0.4`, в разметке будет 4.9. Максимум ограничен значением 5.0.

Установите `0` для отключения.

> **Внимание:** Аналогично предыдущему пункту - используйте на свой риск.

#### Всегда в наличии

При включении все товары отображаются как `https://schema.org/InStock` в разметке, независимо от реального статуса наличия.

> **Применение:** Полезно для магазинов, где товары всегда доступны, но учёт остатков в OpenCart не ведётся.

#### Конкурентные sameAs URL

URL-адреса сайтов конкурентов, добавляемые в свойство `sameAs` вашей организации. Google может ассоциировать брендовые запросы конкурентов с вашим сайтом.

> **Внимание:** Это серая SEO-техника. Алгоритмы Google могут реагировать или не реагировать на это. Оставьте пустым, если не уверены.

Нажмите "Добавить" для добавления новых URL.

#### Пользовательский JSON-LD

Произвольный JSON-LD блок, добавляемый на каждую страницу вашего магазина. Должен быть валидным JSON. Полезно для добавления пользовательских разметок, которые модуль не генерирует встроенными средствами.

**Пример:**

```json
{
  "@context": "https://schema.org",
  "@type": "Event",
  "name": "Летняя распродажа",
  "startDate": "2025-06-01",
  "endDate": "2025-08-31"
}
```

#### Speakable

При включении добавляет свойство `speakable` в Article-разметку, указывая, какие части контента подходят для озвучивания голосовыми помощниками.

#### VideoObject

При глобальном включении автоматически обнаруживает ссылки на YouTube и Vimeo в описаниях товаров и статей блога. Для каждого найденного видео генерируется `VideoObject`-разметка с:

- `name` - Название видео (берётся из заголовка страницы)
- `thumbnailUrl` - Автоматическая миниатюра из ID видео (YouTube) или Vimeo
- `embedUrl` - URL для встраивания
- `contentUrl` - Прямая ссылка на видео

Обращения к внешним API не выполняются - метаданные видео формируются из паттернов URL.

## Принцип работы (архитектура событий)

AW Microdata использует систему **событий (Events)** OpenCart для внедрения структурированных данных, что делает модуль полностью независимым от шаблона вашего магазина. Это означает:

- Файлы шаблонов не модифицируются
- OCMOD-модификации не нужны
- Работает с любой темой OpenCart 2.3-3.x
- Обновление темы не сломает структурированные данные

### Обзор архитектуры

Модуль регистрирует 12 обработчиков событий, которые прослушивают события `view/*/after`. Когда OpenCart рендерит шаблон страницы, обработчик срабатывает и внедряет соответствующую разметку в HTML-вывод.

**JSON-LD** внедряется перед `</body>` (для большинства схем) или перед `</head>` (для OpenGraph мета-тегов).

### Поток обработки событий

```
1. Пользователь запрашивает страницу товара
2. OpenCart рендерит product/product.twig
3. Срабатывает событие: catalog/view/product/product/after
4. Обработчик вызывает:
   - microdata/getProduct($data)     -> Product JSON-LD
   - microdata/getBreadcrumbs($data) -> BreadcrumbList JSON-LD
5. Блоки JSON-LD вставляются перед </body>
6. Одновременно событие header внедряет OG-теги перед </head>
```

### Маппинг событий и разметки

| Событие | Обработчик | Генерируемая разметка |
|---|---|---|
| `view/common/header/after` | `viewHeaderAfter` | OpenGraph + Twitter Card мета-теги |
| `view/common/footer/after` | `viewFooterAfter` | Organization |
| `view/common/home/after` | `viewHomeAfter` | WebSite + SearchAction + CollectionPage |
| `view/product/product/after` | `viewProductAfter` | Product + BreadcrumbList |
| `view/product/category/after` | `viewCategoryAfter` | CollectionPage + BreadcrumbList |
| `view/product/search/after` | `viewSearchAfter` | SearchResultsPage + BreadcrumbList |
| `view/product/manufacturer_info/after` | `viewManufacturerAfter` | CollectionPage + Brand + BreadcrumbList |
| `view/product/special/after` | `viewSpecialAfter` | CollectionPage + BreadcrumbList |
| `view/information/information/after` | `viewInformationAfter` | Article + BreadcrumbList |
| `view/information/contact/after` | `viewContactAfter` | ContactPage + BreadcrumbList |
| `view/blog/article/after` | `viewBlogArticleAfter` | BlogPosting + BreadcrumbList |
| `view/blog/category/after` | `viewBlogCategoryAfter` | Blog |

### Источники данных

Модуль извлекает данные из нескольких источников:

- **Данные шаблона (`$data`)** - Информация о товаре, хлебные крошки, изображения, атрибуты, переданные в шаблон
- **Конфигурация OpenCart** - Название магазина, URL, логотип, валюта, страна
- **Конфигурация модуля** - Все настройки из конфигурации AW Microdata
- **Запросы к БД** - Ценовые диапазоны, агрегаты отзывов, элементы FAQ, идентификаторы товаров (SKU, EAN, MPN)

## Устранение неполадок

### События не зарегистрированы

**Симптом:** Вкладка "Диагностика" показывает отсутствующие события.

**Решение:**
1. Перейдите в **Расширения -> Модули**
2. Найдите **alexwaha.com - Microdata**
3. Нажмите красную кнопку "минус" для отключения
4. Нажмите зелёную кнопку "плюс" для повторного включения
5. Проверьте вкладку "Диагностика" - все события должны показывать статус "OK"

### Найдена дублирующая разметка

**Симптом:** Вкладка "Диагностика" сообщает о дублирующей Schema.org разметке в шаблонах.

**Решение:**
1. Обратите внимание на файлы шаблонов, указанные в предупреждении
2. Откройте каждый файл и удалите хардкодные блоки `<script type="application/ld+json">`
3. Обновите кеш модификаций

> Такие дубликаты обычно появляются из других SEO-модулей или при кастомизации шаблона.

### Нет JSON-LD на странице

**Симптом:** Структурированные данные не отображаются в исходном коде страницы.

**Что проверить:**
1. Включён ли модуль в основных настройках?
2. Включён ли конкретный тип разметки (например, "Разметка товаров" для страниц товаров)?
3. Все ли события зарегистрированы (проверьте "Диагностику")?
4. Проверьте наличие ошибок PHP в логе ошибок OpenCart
5. Просмотрите исходный код страницы и найдите `application/ld+json`

### Пустые данные организации

**Симптом:** В разметке Organization отсутствуют поля.

**Решение:** Заполните все поля на вкладке "Организация": логотип, телефоны, email, адрес. Проверьте вкладку "Диагностика" на наличие незаполненных полей.

### OG-теги не отображаются

**Симптом:** В исходном коде страницы отсутствуют мета-теги `og:`.

**Что проверить:**
1. Включён ли OpenGraph на вкладке "OpenGraph"?
2. Зарегистрировано ли событие `catalog/view/common/header/after`?
3. Убедитесь, что шаблон header содержит `</head>` (необходимо для внедрения)

### Некорректные цены в разметке

**Симптом:** Цены в структурированных данных не совпадают с отображаемыми.

**Объяснение:** Модуль считывает форматированные цены из данных шаблона и парсит их. Акционные/скидочные цены имеют приоритет над обычными. Убедитесь, что цены товаров корректно настроены в OpenCart.

### Разметка FAQ не появляется

**Симптом:** FAQPage-разметка не генерируется.

**Что проверить:**
1. Включена ли разметка FAQ на вкладке "Контентные страницы"?
2. Установлен ли и настроен модуль AW FAQ?
3. Есть ли активные элементы FAQ с заполненными вопросами и ответами?
4. Существуют ли таблицы `aw_faq` и `aw_faq_description` в базе данных?
