Instruction | Alexwaha.com - SMS notifications for Opencart v2.3 -3.x

Before installing the extension, watch the YouTube video in Russian - YouTube link

Installation instructions

This extension contains a module and SMS gateways for popular messaging services

  • 1. Make sure you have the awCore-oc2-3.ocmod.zip module installed This is a helper functionality for all my modules
  • 2. Make sure you have a module installed that disables FTP-loading of extensions in Opencart 2.x or you have correct FTP settings in your store settings - Link to the module disabling FTP-loading only for Opencart v2.3 (Not required for Opencart 3.x)
  • 3. Install the aw_sms_notify_oc_2_3.ocmod.zip archive through the admin panel
  • 4. Update the modification cache
  • 6. In the section: Extensions -> Modules -> Enable the module - alexwaha.com - SMS notifications When enabled, the module will automatically delete old version files, install Events and Permissions

Self-check after installation:

  • Check if the module is in the extensions list
  • Check for the following values in Extensions - Events: aw_sms_notify_order_alert, aw_sms_notify_review_alert, aw_sms_notify_register_alert
  • Update the modification cache
Gateway configuration

Go to the module settings alexwaha.com - SMS notifications

  • On the tab "Gateway settings" set the necessary settings to connect to the SMS gateway.
  • SMS gateway - Select a gateway from the list of available ones
  • Sender - Specify the registered Sender Name (Alpha name) 11 Latin characters without spaces. Registered directly in your messaging service personal account
  • Administrator phone number - Specify the phone number in international format if notifications are needed about new orders or new product reviews for example: +1(operator code) 1234567
  • Additional numbers - Numbers to which copies of notifications will be sent (same as for administrator). Enter numbers separated by commas (without spaces) in international format +1(operator code) 1234567
  • SMS gateway login (or API KEY) - Login or API key to connect to the messaging service You can find this information in the personal account of the messaging service you use
  • SMS gateway password - Password for the SMS gateway If an API key is used to connect to the gateway, the password can probably be left empty.
Logging settings

On the tab "Gateway logs".

  • 1. You can specify a custom name for the log file
  • 2. The logs will store information about sends, errors and other things related to the selected gateway for sending SMS.
Notification settings

On the tab "Notification settings" enable only the necessary notifications.

  • Send SMS to admin - notification to the administrator about a new store order to the phone numbers specified earlier in the "Administrator phone number" and "Additional numbers" fields.
  • Send SMS to customer - notification to the customer about a new order to the number specified in the order. Below in the "SMS for customer groups" block, you need to specify which customer groups will receive the notification.
  • SMS to customer upon registration - notification to the customer about registration. Allows you to send login data.
  • SMS when order status changes - notification about order status change by manager, delivery module or automatically.
  • SMS for new reviews - notification to the administrator when a new product review is received.
  • Force SMS sending - Forced sending of notifications. Necessary if requests to send notifications about order status changes will be made by third-party extensions.
  • SMS text transliteration - Converting Cyrillic to Latin. To reduce the number of SMS sent, it's better to use transliteration. Since in Cyrillic 1 SMS contains 70 characters, in Latin 140 characters. Important! To avoid sending errors, do not enable this parameter if you already specify text in Latin in the templates.
  • SMS for customer groups - Automatic sending of notifications only to selected customer groups. If no groups are marked, notifications will be sent to all customers.
  • SMS for payment methods - Automatic sending of notifications only for selected payment methods after order placement. With notification sending set for payment methods, the notification for the customer for a new order will be replaced with a notification for the selected payment method in the order.
  • SMS for statuses - Automatic sending of notifications for selected order statuses when the order status changes.
  • SMS templates when viewing an order - Here you can specify frequently used texts for notifications when sending notifications manually by the manager from the form when viewing an order.
Viber notification settings

On the tab "Viber Settings - Notification Settings" if necessary, you can configure sending notifications to Viber.

  • Send Viber messages - Enable/Disable sending to Viber.
  • Viber sender name - Specify the registered Viber Sender name or common name provided by your messaging service. Viber sender name is displayed directly in the personal account of the messaging service used and is registered in advance
  • Viber message lifetime - Time in seconds during which the message in Viber will be delivered. If the Viber notification is not delivered within the specified time period, or there is no Viber account on the specified phone number, an SMS message will be sent.
  • Image in Viber message - The image that will be displayed in each Viber message. You can specify the store logo. The default image height and width is 400*400px, you can specify proportionally larger, but the image must be square.
  • Button text in Viber message and Button link - Parameters that need to be specified if you want to show a button with a link in the Viber message. If these parameters are not filled in, the button will not be displayed in the message.
Setting up customer notification templates

On the tab "SMS template to customer" you set up SMS notification texts using text and variables from the "Variables" tab.

  • SMS template for registration - a notification sent to the customer upon registration. SMS is sent when a new order is placed, if the parameter "Send SMS to customer" is enabled and for an order where payment method has not yet been assigned status. i.e. order_status_id = 0.
  • SMS template to customer (new order) - a notification sent to the customer about a new order. SMS is sent when a new order is placed, if the parameter "Send SMS to customer" is enabled and for an order where payment method has not yet been assigned status. i.e. order_status_id = 0.
  • SMS for payment methods - If a template is set up and SMS sending for payment methods is enabled, then the New Order template for the user will be ignored. The SMS message will be replaced with the SMS specified in the payment method template chosen at the order.
  • SMS templates for order statuses - If a template is set up and SMS sending for status changes is enabled, then the corresponding template for the established status will be used.
Setting up administrator notification templates

On the tab "SMS template to customer" you set up SMS notification texts using text and variables from the "Variables" tab.

  • SMS template for administrator (new order) - a notification about a new order to the administrator. SMS is sent when a new order is placed, if the parameter "Send SMS to admin" is enabled.
  • SMS template for new reviews - a notification about a new review on a product to the administrator. If a template is set up and sending of SMS "SMS for new reviews" is enabled. Allowed tags {{product.name}}, {{product.model}}, {{product.sku}}, {{product.date}}. Product name is truncated to 50 characters.
Variables

"List of variables" variables that are allowed to be used in notification templates.

Variable {{products}} is not specified directly. To use it, you must follow the strict syntax of the TWIG-template parser, as shown below:

{% for product in products %}

Product: {{product.name}}

Model: {{product.model}}

Price: {{product.price}}

Quantity: {{product.quantity}}

{% endfor %}

For Developers

With the module alexwaha.com - SMS notifications you can send messages from other sources.

For this, you need to connect the model of the SMS notification module (catalog directory only) in your code by example:

Model load for Opencart 2.3.x - Opencart 3.x

$this->load->model('extension/module/aw_sms_notify');

Preparing data for sending:

$smsData = [ 'phone' => '+1234567890', 'message' => 'SMS message text' ];
Where, phone - International phone number format; message - Message text generated by your code

Sending data to the model:

$this->model_extension_module_aw_sms_notify->sendMessage($smsData);

Returns bool, true - Data is correct and sent to the sendMessage method, false - if phone or message are missing.

Actual SMS sending is visible in the module's log and in the SMS service panel

Possible errors and recommendations
If no SMS are sent

Check your balance in the service of sending.

Check the correctness: login and password. Often, the login and password for connecting to the SMS gateway differ from the access to the Personal Cabinet in the SMS service. What to indicate in the fields Login and Password in the settings of the SMS notification module you can learn on the tab "List of ready integrations/slaves" of this guide. Opposite the necessary slave will be indicated, which protocol is used for connection and with what authorization (Login/Password/API key, Token) is performed.

Check the logs of the gateway. If the connection to the gateway is set up correctly, the gateway will record possible errors in the logs. Decoding error codes can be found on the website of the used service for sending or when contacting their technical support.

Check the registration of Alpha-name/Signature sender. Perhaps the signature has not been approved or you have not registered it yet. When specifying an unknown signature in the log, errors may occur, and SMS will not be accepted by the gateway. I strongly recommend not specifying a Sender if you do not know him!

If no SMS is sent when changing the order status

Check the settings of the SMS notification module. SMS when changing order status should be enabled, selected statuses for which sending is required, SMS templates for selected statuses are set. And in the Extensions - Events section, there are records aw_sms_notify_order_alert Events exist in versions Opencart 2.3.x - 3.x

Perhaps the management of orders is used by a third-party module. For example Order Management from Newstore, OrderPro, Order Manager Pro etc. In this case, you need to install additional integrations, which can be found in the folder sms-notification/dist/3rd-party!

If when trying to send SMS, an error window appears or with a text that is not clear to you

Perhaps the chosen in the settings of the module gateway uses SOAP protocol, if it is used, then you need to install SOAP extension on the server.

If when trying to send SMS, your white screen or completely freezes the order form page, it's better to turn off SMS sending and contact by contacts at the bottom of this guide.
